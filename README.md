# 秘密家园

一个单文件的 AI 对话前端，用来在浏览器里管理多个 AI 助手、多个对话、记忆条目、模型供应商和本地/云端备份。由Claude、ChatGPT辅助完成。

它的目标不是做成复杂平台，而是提供一个轻量、私密、可自行托管的 AI 伴侣/角色对话界面。

## 功能概览

* 多助手管理：可以创建不同助手，分别设置名称、头像、颜色、模型、System Prompt 和参数。
* 双模式界面：支持普通角色扮演模式，也支持类似即时通讯的聊天模式。
* 多模型供应商：支持维护多个 OpenAI 兼容接口，也内置 OpenRouter、SiliconFlow、DeepSeek 等配置入口。
* 自定义模型：可以选择模型列表里的模型，也可以手动填写模型 ID。
* 记忆库：可以添加带关键词的记忆条目，在对话中自动注入相关内容。
* 对话管理：支持新建、重命名、删除、搜索消息，以及批量删除消息。
* 图片输入：支持在对话中发送图片。
* 缓存与统计：显示缓存状态、token、成本估算、使用统计等信息。
* 本地保存：对话和设置保存在浏览器本机。
* 本地备份：可以导出 JSON 备份文件，也可以从本地备份恢复。
* WebDAV 备份：支持配置 WebDAV，将数据备份到云端。
* Kelivo 导入：支持从 Kelivo 的 chats JSON 文件导入对话。
* PWA 支持：可以添加到手机桌面，作为类 App 使用。
* VPS 伴侣接口：预留了连接 VPS 的入口，可用于服务端对话和主动通知等扩展功能。

## 文件说明

这个项目最核心的文件是：

```text
index.html
```

它包含了页面结构、样式和主要逻辑。也就是说，只要上传这个文件，页面本身就可以运行。

可选文件：

```text
icon.PNG
```

用于手机添加到桌面时显示图标。如果不放这个文件，页面仍然可以使用，只是图标可能不是自定义图标。

```text
sw.js
```

用于 PWA 推送通知。如果只是普通聊天，不需要这个文件；如果要使用 VPS 主动通知功能，则需要额外配置。

## 使用方法

### 1. 下载或上传文件

把 `index.html` 放到你的 GitHub 仓库里。

如果你有自定义图标，也可以把 `icon.PNG` 放在同一个目录下。

### 2. 开启 GitHub Pages

在 GitHub 仓库页面中进入：

```text
Settings → Pages
```

然后选择部署分支，例如：

```text
main / root
```

保存后，GitHub 会生成一个网页地址。

### 3. 打开网页

用浏览器打开 GitHub Pages 生成的地址，就可以开始使用。

第一次使用时，需要在页面设置里填写模型供应商、API Key 和模型信息。

## 基础配置

进入页面后，点击右上角设置按钮，可以配置：

* 模型供应商
* API Key
* Endpoint / Base URL
* 模型列表
* 时间戳注入
* 聊天模式格式说明
* WebDAV 备份
* VPS 伴侣接口

每个助手也可以单独设置：

* 名称
* 头像
* 颜色
* API Provider
* 模型
* 自定义模型 ID
* Temperature
* 缓存策略
* 回复长度上限
* 上下文消息数
* 是否启用思考链
* System Prompt

## 数据保存与隐私

这个项目默认把数据保存在浏览器本机。

主要包括：

* 助手设置
* 对话记录
* 记忆库
* 模型供应商配置
* API Key
* 本地备份相关配置

这些数据不会因为你把 `index.html` 上传到 GitHub 而自动公开。

需要注意的是：

* 不要把自己的 API Key 写死进 `index.html` 代码里。
* 不要把导出的备份 JSON 文件上传到公开仓库。
* 不要把包含私人 System Prompt、聊天记录或密钥的截图上传到公开仓库。
* 如果开启 WebDAV 备份，云端备份文件由你配置的 WebDAV 服务保存。

## 备份方式

### 本地备份

可以在页面中点击本地备份按钮，导出 JSON 文件。

这个文件包含你的对话、助手、记忆和相关配置。请妥善保存，不建议公开上传。

### WebDAV 备份

如果你配置了 WebDAV，可以把数据同步到自己的云端空间。

部分 WebDAV 服务可能需要 CORS 代理，否则浏览器无法直接连接。

### 恢复备份

恢复备份时有两种方式：

* 完全覆盖：用备份数据替换当前本地数据。
* 智能合并：保留当前已有数据，只补充备份中有但本地没有的部分。

恢复前建议先导出一次当前数据，避免误操作。

## 关于 VPS 伴侣功能

页面中预留了 VPS 接口配置，可以连接到你自己的服务器。

这个功能适合做更进一步的扩展，例如：

* 让服务器调用模型
* 保存服务器端 System Prompt
* 主动唤醒
* PWA 通知
* 服务端上下文管理

如果只是普通网页聊天，不需要配置 VPS。

## 部署建议

如果只是自己使用，推荐：

```text
GitHub Pages + 浏览器本地保存 + 本地备份
```

如果需要跨设备同步，推荐：

```text
GitHub Pages + WebDAV 备份
```

如果需要主动通知或服务端能力，推荐：

```text
GitHub Pages + VPS + sw.js + HTTPS 域名
```

## 开源注意事项

如果准备公开仓库，建议确认以下内容没有被提交：

* API Key
* 备份 JSON
* 私人聊天记录
* 私人 System Prompt
* VPS 连接密钥
* 真实私人信息
* 不想公开的角色名称或设定

`index.html` 本身可以作为前端源码公开，但私人数据应该只留在浏览器本地或自己的备份文件里。

## 浏览器兼容

推荐使用最新版 Chrome、Edge、Safari 或手机系统自带浏览器。

由于项目使用了浏览器本地存储能力，隐私模式、无痕模式或清理浏览器数据可能会导致本地数据丢失。使用前建议定期导出备份。
